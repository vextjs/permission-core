# v1.0.0 — 正式发布

**发布日期**：2026-05-14  
**发布类型**：Major（首个正式稳定版本）

---

## 概览

permission-core v1.0.0 是该库的第一个正式稳定版本。在 v0.2.0 运行时基线之上完成了以下收口工作：

- **实际 HTTP 集成验证**：通过 vext 框架启动真实 HTTP 服务器，25 项端到端测试全部通过
- **覆盖率 100%**：语句、分支、函数、行四项全达 100%（完整单测集通过）
- **文档站补齐**：首页 features 扩展为 6 项，含角色继承与行级权限说明
- **自动化部署**：配置 GitHub Pages workflow，push to main 即自动构建发布文档站

---

## 新增

### 集成测试（vext 框架）
- `src/rbac/role-manager.ts` — 新增 `getRoleChain()`、`getEffectiveRules()`、`inspect()`，直接支持角色详情页 / 调试面板查看 own rules、effective rules 和继承链
- `vext-test/src/plugins/permission.ts` — 注册 PermissionCore 到 vext 应用
- `vext-test/src/routes/permission.ts` — 暴露 7 个 HTTP 端点（check / resources / assert / roles CRUD + 角色权限检查）
- `vext-test/verify-permission-http.mjs` — 26 项 HTTP 集成验证脚本，覆盖角色允许/拒绝、多角色合并、通配符、角色继承、行级 assert、roles CRUD 与角色 effective rules 检查

### CI/CD
- `.github/workflows/pages.yml` — GitHub Pages 自动部署 workflow（Node 22 + rspress build + deploy-pages）

### 文档
- 首页 features 新增"角色继承与规则合并"和"细粒度行级与字段权限"

---

## 修复

- vext 集成：`app.throw(403, message, code)` 代替 `res.status(403).json()` 以符合框架错误格式规范

---

## 不变

- 公开 API 完全向后兼容 v0.2.0
- `StorageAdapter` 抽象接口不变
- 既有运行时鉴权 API 与 `getRules()` 语义不变；角色 inspection 能力以新增 API 形式提供
