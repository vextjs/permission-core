# permission-core v0.2.0

> 发布日期：2026-05-13
> 发布类型：MINOR
> 基线提交：`988c5c3` `feat: implement permission-core runtime`

## 概览

`v0.2.0` 是 permission-core 从“文档与工程骨架”进入“完整可用实现”的首个正式发布版本。该版本把统一权限模型、运行时鉴权、角色规则管理、三种存储适配器、行级权限 API、自动化验证和独立文档站一次性收口为可追踪的发布基线。

## 新增能力

### 核心运行时

- 新增 `PermissionCore` 与 `PermissionCoreContext`，统一提供 `can`、`assert`、`getPermissions`、`getResources`、`getRowScope`、`filterRows`、`filterFields` 等能力。
- 新增 `Resolver` 与 `Checker`，实现角色继承展开、allow/deny 规则判定、`write` 双向语义和资源匹配。
- 新增 `matchResource` 子路径导出，用于独立复用资源匹配能力。

### RBAC 与存储

- 新增 `RoleManager` 与 `UserRoleManager`，支持角色管理、规则维护、用户角色绑定与缓存失效。
- 新增 `MemoryAdapter`、`FileAdapter`、`MonSQLizeStorageAdapter` 三类存储适配器。
- 接入 `cache-hub` 作为缓存底座，并实现基于用户和规则变更的缓存失效策略。

### 数据权限

- 新增 `RowCondition` DSL，支持 `all`、`any`、`not` 组合与 `eq`、`in`、`gt`、`contains`、`exists` 等条件运算。
- 新增行级判定与字段过滤链路，支持在 DB-only 和 Full standard stack 场景下收口记录范围与返回字段。

### 文档与示例

- 新增独立 `website/` 文档站，覆盖指南、API 和示例三大入口。
- 新增实现阅读顺序、管理后台保存、行级权限、MonSQLize 适配器等专题内容。
- 更新仓库根 README，使仓库入口与文档站路径一致。
