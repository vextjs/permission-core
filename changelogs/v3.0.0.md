# v3.0.0

Release date: 2026-07-20

## Added

- Added `scoped.menus.config.preview/save/get/list/previewRemove/remove/previewChanges/applyChanges` as the business entry for backend menu configuration.
- Added `MenuConfigInput` with menus, views, load APIs, actions, `api:METHOD:/path` resources, and response-field definitions.
- Added persisted schema 3 menu config documents, aggregate compilation, config audit evidence, and schema 2 rejection for this release line.
- Added role-menu business grants for selected menus, views, loads, actions, and response fields, including direct, effective, and authorization-tree reads.
- Added subject menu runtime projection with `getViewTree()`, `getActionMap()`, `getViewState()`, and `filterResponse()`.
- Added Vext request integration for permission context helpers, route guards, response projection, and protected route cache fail-closed behavior.

## Changed

- Removed the old advanced public menu/API-binding surface from the intended package contract: raw menu manifest management, public `scoped.apiBindings`, bottom-level API owner selections, stale-source repair helpers, and Vext binding conversion helpers are no longer user-facing.
- Kept MonSQLize 3.1 as the only persistence contract; no `StorageAdapter`, cache-hub direct integration, or migration guide is included for this unreleased major redesign.
- Standardized documentation and examples on `api:METHOD:/path` resources for backend interface permissions.
- Updated `examples/menu-admin.mjs` to show the full menu config lifecycle: preview, save, grant, bind a user, read runtime state, and project response fields.
- Updated README, examples, API docs, and guide pages for the new business-first menu model.

## Fixed

- Closed the documentation drift that mixed old `nodes/apiBindings/getButtonMap` terminology with the new config/runtime APIs.
- Clarified that `cannot()` is the negation of `can()` and does not assign blocked permissions.
- Hardened Vext protected responses with `private, no-store` cache control for user-specific authorization output.
